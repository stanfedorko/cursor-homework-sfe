• git config --global user.name blabla (Задати імя користувача)
  • git config --global user.email blabal@bla (Задати емейл користувача)
  • git init (створює папку .git в якій зберігаються файли репозиторія, зміни тощо)
  • git status (показує стан файлів в робочій папкі, які файли змінені і не добавлені або які очікують відправки)
  • git add (вибераються файли які будуть комітиться)
  • git commit (зберігає всі файли що ми вибрали в коміт)
  • git log (перегляд історії комітів)
  • git checkout (для переключення гілок та вигрузку в робочу папку)
  • git branch (керування гілками, створення, перегляд, видалити і тд)
  • git push (передача файлів "змін" на відділений репозиторій)
  • git pull (заберає зміни з віддаленого репозиторію)
  • git merge (злиття гілок)
  • git clone (команда використовується для клонування "копіювання" репо)

.gitignore - це файл в якому вказуються папки та файли які Git пропускає і не контролює "не відслідковує" їхні зміни.

  • git revert (відміняє коміт, відкатує, створює новий коміт без змін)
  • git reset (відміняє зміни)
  • git rebase (обєднує зміни зроблені в одній гільці з іншою гілкою)
  • git cherry-pick (забрати коміти з однієї гілки в іншу)
  • git stash (ховає зміни в файлах і зберігає їх окремо, потім можна їх повернути)
